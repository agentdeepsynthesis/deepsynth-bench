## Evaluation

### Metrics

We provide three evaluation metrics:

| Metric | Description |
|--------|-------------|
| **Exact Match (EM)** | All keys and values must be exactly correct |
| **F1 Score** | Partial credit for correct key-value pairs |
| **LLM Judge** | Soft evaluation allowing semantic equivalence and small numerical margins (1-5.5%) |

### Running Evaluation

```bash
pip install -r requirements.txt

# Evaluate model predictions
python evaluation/evaluate.py \
  --predictions predictions.json \
  --ground_truth DEEPSYNTH_lite.json \
  --metrics f1,em,llm_judge
```

### Prediction Format

Model predictions should be a JSON file with task IDs mapped to answers:

```json
{
  "001": {"Sweden": 1.2, "Finland": 0.8},
  "002": {"Brunei": -0.67, "Singapore": -0.34},
  ...
}
```

## 🗂️ Auxiliary Resources

### Question Data (`files_to_answer_the_questions/`)
Some questions require external data or scripts to be solved. These are located in the `files_to_answer_the_questions` directory.
* The subdirectories are named by question ID (e.g., `001`, `002`).
* Each folder contains the specific files relevant to that question number.

## 🧩 Decompositions & Validation

This section handles the breakdown of complex questions into manageable sub-problems.

### 1. Decomposition Files (`decompositions/*.json`)
Selected questions have corresponding JSON files in the `decompositions/` directory. These files map the logical steps to solve a problem.
* **Core files:** `001.json`, `002.json`, etc.
* **AI Resources:**
    * `gpt_generated_decompositions.json`: Decompositions generated by GPT.
    * `intermediate_answers_prompt.txt`: Prompt template for generating answer scripts.

### 2. Validation Schemas (`decompositions/intermediate_answers_schemas/`)
To ensure data integrity, each decomposition file has a matching JSON Schema located in this directory (e.g., `001.schema.json`). These schemas define the expected format for the `intermediate answers` field.

## 🛠️ Scripts

The `scripts/` directory contains Python utilities for maintaining and expanding the benchmark:

* **`format_jsons.py`**: Standardizes the formatting of JSON files in the repository (applies indentation and pretty-printing).
* **`validate_intermediate_answers.py`**:
    * **Schema Validation:** Validates the `intermediate answers` field in `decompositions/*.json` against the corresponding schema in `decompositions/intermediate_answers_schemas/`.
    * **Consistency Check:** Ensures that the number of items in the `decomposition` list matches the number of items in the `intermediate answers` list.
* **`gpt_decompositions_schemas.py`**: A script that prompts GPT to automatically decompose questions into sub-steps and generate the corresponding JSON Schemas for the answers.
